# Documentation Images

Images in `docs/img/` are committed to the repository so the docs render correctly on GitHub without a build step. Regenerate them after changing the heatmap algorithm or the scenes in the test.

## Peaks algorithm diagram (`docs/img/peaks_diagram.png`)

Generated by `TestPeaksDiagram` in `docs/gen/peaks_diagram_test.go`.

```bash
go test ./docs/gen/ -run TestPeaksDiagram -v
```

Skipped under `-short`. Re-run after changing the `Peaks()` algorithm (bucket split, PCA, peak spacing).

The diagram shows a synthetic elongated hot component (diagonal heatmap band), the two dashed bucket boundaries, and the three yellow peak circles. Axes and layout are hard-coded in the test; the peaks themselves are drawn at fixed fractions (1/6, 1/2, 5/6) of the axis for clarity — they do not invoke `Peaks()`.

## Kernel comparison images (`docs/img/kernel/`)

Generated by the test `TestKernelComparison` in `docs/gen/kernel_compare_test.go`.

```bash
go test ./docs/gen/ -run TestKernelComparison -v
```

The test is skipped in the normal test suite (`make test` / `make check` both pass `-short`) to avoid rewriting files on every CI run. Run it explicitly after touching any of the following:

- `internal/heatmap/heatmap.go` — kernel formula, sigma, aggregation, or normalisation
- `docs/gen/kernel_compare_test.go` — synthetic scene coordinates or similarity values

### Data sources

The **aggregation comparison** (`agg_compare.png`) uses a fully synthetic scene.

### Synthetic scene (`complexFeatures`) — aggregation only

The aggregation comparison uses this scene (σ=300 m, 75 m/cell):

```
bbox 0.0°–0.10° × 0.0°–0.10°  (≈ 11 km × 11 km at the equator)

Park polygon   (upper-left)                          sim=0.45
River          (diagonal through the park and beyond) sim=0.45
Canal A        (horizontal, y=0.024)                  sim=0.65
Canal B        (horizontal, y=0.029, gap ≈ 556 m)    sim=0.65
Tight cluster  9 points within a 157 m radius         sim=0.55
               centred at (0.075°, 0.047°)
Loose scatter  7 points, upper-right quadrant         sim=0.55
               (two ≈700 m pairs + three isolated)
```

Key overlap zones:

- **Canal gap** — midpoint between the two canals (≈278 m from each, sim=0.65): canals set the ceiling in max mode at 0.65⁴ ≈ 0.18; the gap fills in sum mode.
- **Tight cluster** — the most dramatic element: 9 points within 157 m of each other.
  - max: peak = 0.55⁴ ≈ 0.09 → 50 % of ceiling → pale yellow dot
  - sum: ~9 contributions accumulate to ≈ 0.62 → 3.5× ceiling → large red circle
- **Park + river** — inside the polygon the river adds a second contribution in sum mode.
- **Loose scatter pairs** — the ≈700 m pairs (< cutoff 900 m) grow slightly in sum mode.

The aggregation comparison uses a **shared ceiling** (95th percentile from the max grid) so that sum-mode accumulation appears as genuinely brighter colours rather than being renormalised away.

### Similarity weight scene (`richmondFeatures`)

Used for the `weight_compare.png` series (same real OSM scene as kernel and sigma comparisons). The Thames centreline (sim=0.95) and parks (sim=0.80) provide a wide enough range of similarity values to make the exponent's effect on relative brightness easy to read.

### Real OSM scene (`richmondFeatures`, `rcRoads`, `rcPaths`)

Used for `kernel_compare.png`, `sigma_compare.png`, and `weight_compare.png`. Fetched via Overpass API and simplified with RDP (ε ≈ 10–12 m); coordinates hardcoded in the test (no network access needed at test time).

```
bbox  lon [-0.317, -0.284]  lat [51.448, 51.468]  (≈ 2.3 km × 2.2 km)

Heatmap features (scored):
  Thames centreline   LineString, OSM way 26456375, 11 nodes   sim=0.95
  7 parks             Polygons (Old Deer Park, Marble Hill,
                      Terrace Gardens …)                        sim=0.80
  8 viewpoints/       Points (Richmond Hill, Richmond Bridge…)  sim=0.75
    monuments
  14 cafés/pubs       Points                                    sim=0.55

Context overlay (not scored):
  rcRoads  121 ways  primary / secondary / tertiary / pedestrian
  rcPaths  250 ways  footways and paths (longest 250 by length)
```

Resolution 20 m/cell, scale ×3 → ≈345×330 px per tile.
